## 3. 项目进展

当日共合并/关闭 **5 个 PR**，涵盖安全修复、新功能、依赖更新及 UI 改进：

- **#1124 - Add context command support for chat turns** ([PR](https://github.com/moltis-org/moltis/pull/1124))  
  新增 `chat.context_command` 配置，允许在每个聊天轮次前执行命令并将 stdout 追加到提示上下文中。此功能使部署环境可以自动注入运行时上下文，特别适合需要动态生成提示的场景（如注入系统状态、日志片段等）。覆盖了配置 schema、模板、验证和文档。

- **#1162 - fix(web): show dates for older sessions** ([PR](https://github.com/moltis-org/moltis/pull/1162))  
  修复 Issue #1108。Web UI 会话列表现在对今天的会话显示 `HH:MM`，对近期日期显示“昨天”或星期几，对更早日期显示具体日历日期（必要时包含年份）。实现了四个日期桶的完整浏览器覆盖。

- **#1164 - fix(slack): allow operator-approved api base hosts** ([PR](https://github.com/moltis-org/moltis/pull/1164))  
  将 Slack API base URL 验证逻辑移入 `channels` crate，供 gateway 和 Slack 客户端共享。新增 `MOLTIS_SLACK_API_BASE_URL_ALLOWLIST` 环境变量，由运营商控制以允许内部 Slack 代理主机，同时仍阻止云元数据端点。

- **#1163 - fix(slack): challenge unknown allowlist DMs with OTP** ([PR](https://github.com/moltis-org/moltis/pull/1163))  
  修复 Slack 白名单语义：空 DM/channel 白名单现在拒绝访问而非保持开放。为非白名单 DM 用户新增 Slack OTP 自审批流程。同时修复了 Microsoft Teams、Signal 群组访问和 Matrix DM 中的类似空白名单绕过问题。

- **#1161 - chore(deps): bump astro from 7.0.9 to 7.1.3 in /docs** ([PR](https://github.com/moltis-org/moltis/pull/1161))  
  将文档站点的 Astro 框架从 7.0.9 升级到 7.1.3（`npm_and_yarn` 组依赖更新）。

## 4. 社区热点

- **[[Bug] ReAct Agent 上下文（context）中 tool_result 混入 role:assistant 消息 (Issue #6407)](https://github.com/agentscope-ai/QwenPaw/issues/6407)**: 这是一个严重的兼容性问题。用户报告 `ReAct Agent` 在恢复会话时，会将 `tool_call` 和 `tool_result` 混入同一条 `role:assistant` 消息，导致 OpenAI 兼容 API 报 400 错误，模型无法继续执行。该问题直接影响了所有使用 `ReAct` 模式的用户，讨论热度较高。

- **[[Bug] tool_call arguments polluted with markdown fences (Issue #6363)](https://github.com/agentscope-ai/QwenPaw/issues/6363)**: 社区持续关注不同模型（如 GLM、DeepSeek）返回的 `tool_call` 参数格式问题。模型将参数包裹在 Markdown 代码块中，导致 JSON 解析失败，所有工具调用失效。这反映了项目在处理非标准化模型输出时面临的普遍挑战。

- **[[Question] 升级2.0以后，mcp工具总是提示Tool not found (Issue #6405)](https://github.com/agentscope-ai/QwenPaw/issues/6405)**: 升级到 v2.0 后，MCP 工具普遍出现“Tool not found”的错误。用户反馈工具名称的格式变更（`[mcp-key]__[tool_name]`）没问题，但系统仍无法找到工具。此问题影响了大量依赖 MCP 扩展功能的用户，是当前社区体验的一个痛点。
